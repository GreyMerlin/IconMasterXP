Supported Color Modes or Bit-Depths

To change a bit-depth of all currently selected images:

  • Through main menu, click Edit → Bit Depth → [Desired Color Mode]
    Read more bellow on possible color modes.
    - Or -
    Use the right-click menu on the bottom workspace area.

IconMasterXP supports most common image color modes and allows you to convert images both ways. Color mode or bit-depth is a most important attribute of an image and using it properly in your designs greatly improves quality of your work.
Windows® icons in particular usually require you to support all common bit depts so the icon is displayed properly in all modes. For example having a monochromatic version of your icon stored in the multi-paged icon file is a basic requirement for many software applications.

Further bellow is an explanation for each supported bit-depth.

32-Bit, True Color Plus 8-Bit Alpha Channel

This is actually a 24-bit image with 8 bit alpha channel included. When you export an image with this format, it will be detected as 24-bit image, but most programs in Windows XP support alpha channel and will display your image correctly. On the other hand, if certain program that does not support alpha channels attempts to render 32-bit image, it would display it with rendering errors, such as black pixels around that image. In order to avoid this undesirable effect, you should always consider exporting icons that have at least one more bit-depth format besides 32-bit (multi-page icon files, see Exporting)

24-Bit, True Color

This format should only be used when necessary; it is a bitmap with no palette and no alpha-channel. Also, avoid using this format when exporting icons, 8-bit palette is recommended and common amongst icon designers. Also known as, "true color", this format is used usually for images with more than 256 colors.

Since resulting file size is directly proportional to the number of colors used, expect enormous files if you use this format for exporting bitmapped images or icons. Note that, when you work with this format, number of colors is actually infinite, so you do not need to worry about palettes.

Some special effects and features in IconMaster require images to be converted to 24-bit mode first and then converted to any desirable format when done.

8-Bit, Optimized Palette With Error Diffusion

This is well known format and very common in practice. There is usually a dilemma with this format, should you use compressed 24-bit image or standard 8-bit? Answer is both, the good thing about 8-bit image is palette, it is consisted of 256 different colors, and each has their index (place in the palette). If you on the other hand, need more than 256 colors, or your image is colossal — give chance to JPEG compression. When exporting Windows® icons, this format is usually a must.

IconMaster has built-in color dithering system (palette optimizer with error diffusion) so it will reduce number of colors without harming image quality.

4-Bit, Standard VGA With Error Diffusion

This 16-color format is used only for VGA graphic mode. It has palette consisted of 16 fixed colors which cannot be changed. This format is also used by memory-friendly software, although it is now obsolete it is still used in certain programs with extremely large number of graphical elements, in order to reduce memory-usage and performance strain.
Bellow is a comparison between original image on the left and an 4-bit (with 16 colors) version of the same image on the right.

Notes

  1. Creative designers can draw very functional, clear and good-looking icons with only 16 colors
  2. This was a default color mode of old 16-bit operating systems (eg. Windows® 3.11).

1-Bit, Monochrome (2 Colors) With Dithering

This is monochromatic mode, which means it has no actual colors - only black and white. It is used by special software for better compatibility. Lately however, minimalistic design styles have returned monochrome icons back into fashion.
